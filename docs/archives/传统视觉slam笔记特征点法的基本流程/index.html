传统视觉SLAM笔记：特征点法的基本流程

我现在对传统视觉 SLAM 的理解是：相机一边走，一边回答两个问题，我在哪里，我看过的环境长什么样。输入可以是单目、双目或 RGB-D 图像，输出一般是相机轨迹和地图。特征点法不试图解释每个像素，它会先挑出一些比较稳定、比较容易重复匹配的局部特征，再用这些特征建立跨帧约束。

整体流程#

一个典型的特征点法视觉 SLAM 系统，先按下面几个模块理解：

  1. 前端跟踪：处理当前帧，提取特征点，和上一帧或局部地图匹配，估计当前相机位姿。
  2. 局部建图：选择关键帧，三角化新的地图点，维护局部地图，并做局部 Bundle Adjustment。
  3. 后端优化：把关键帧位姿和地图点放到图优化问题中，最小化重投影误差。
  4. 回环检测：识别当前视角是否回到过去到过的地方，建立长程约束。
  5. 全局优化：通过位姿图优化或全局 BA 修正累计漂移。

ORB-SLAM 系列基本就是沿着这个结构做工程化拆分。它的核心不是某一个单点技巧，而是把特征提取、匹配、关键帧、局部地图、回环和优化组织成一个完整实时系统。

图像输入和相机模型#

视觉 SLAM 的第一步是理解图像和相机之间的几何关系。相机把三维世界点投影到二维图像平面，这个投影过程由相机内参和外参决定。

内参描述相机自身成像参数，比如焦距、主点位置、畸变参数。外参描述相机坐标系和世界坐标系之间的位姿关系，也就是旋转和平移。SLAM 要估计的核心变量就是每一帧相机的外参。

如果是双目或 RGB-D，相机可以直接或间接得到尺度。双目通过左右图像视差恢复深度，RGB-D 通过深度传感器直接提供深度。单目 SLAM 最难的地方在于尺度不可观，只靠一台相机无法直接知道场景真实大小，所以单目系统估计出的轨迹和地图通常存在一个未知尺度。

特征提取#

特征点法的前端通常先从图像中提取局部特征。一个好特征应该满足几个条件：

  • 可重复：同一个物理点从不同视角、不同光照下仍然容易被检测到。
  • 可区分：不同点的描述子差异足够大，方便匹配。
  • 高效：可以实时提取和匹配。
  • 分布合理：不要全部集中在图像某个区域。

ORB-SLAM 使用 ORB 特征。ORB 可以理解为 FAST 角点检测加 BRIEF 描述子的工程化组合，并加入方向估计和尺度金字塔。FAST 负责快速找角点，BRIEF 用二进制描述子描述局部图像块，匹配时可以用汉明距离，速度很快。

ORB 特征的优势是实时性好、工程实现成熟、对 CPU 友好。缺点是它仍然依赖局部纹理。如果场景弱纹理、运动模糊、光照变化大，特征数量和匹配质量都会下降。这也是后来直接法、学习法和稠密表示方法想解决的问题之一。

特征匹配#

提取特征后，系统需要判断当前帧中的特征点对应过去哪一帧或地图中的哪个点。常见匹配方式包括：

  • 帧间匹配：当前帧和上一帧匹配，用于短时间跟踪。
  • 关键帧匹配：当前帧和某个关键帧匹配，用于重定位或回环。
  • 地图点投影匹配：把局部地图点投影到当前帧，在投影附近寻找候选特征。

匹配不能只看描述子距离。实际系统通常还会加入几何约束，例如极线约束、投影窗口、尺度一致性、方向一致性。匹配完成后，还需要剔除外点。常见方法是 RANSAC，例如用本质矩阵、单应矩阵或 PnP 模型反复采样，保留符合几何关系的匹配。

记住一点：特征匹配只是给出“可能对应”的观测关系，它肯定会有噪声，也会有错配。SLAM 不能直接相信每一次匹配，而是要把这些匹配继续放进几何模型和优化框架里检查。

位姿估计#

当前帧位姿估计根据输入条件不同有几种情况：

  • 2D-2D：单目相邻帧之间只有图像点匹配，可以通过本质矩阵或基础矩阵估计相对运动。
  • 3D-2D：已有地图点的三维坐标和当前图像二维观测，可以用 PnP 估计当前相机位姿。
  • 3D-3D：双目或 RGB-D 情况下，两帧都能得到三维点，可以估计刚体变换。

在实际 SLAM 系统中，跟踪阶段常用的是 3D-2D。局部地图中已经存在一批三维地图点，当前帧提取特征后和这些地图点匹配，然后通过 PnP 得到当前相机位姿。之后再用非线性优化最小化重投影误差。

重投影误差是传统视觉 SLAM 的核心误差形式。一个三维地图点经过当前相机位姿和相机内参投影到图像上，投影位置和真实观测位置之间的像素差，就是重投影误差。优化的目标就是让所有匹配点的重投影误差尽可能小。

关键帧机制#

如果每一帧都进入后端优化，计算量会非常大，而且大量相邻帧信息重复。关键帧机制就是从视频流中挑出一部分更有代表性的帧，用它们来维护地图和做优化。

关键帧选择一般考虑：

  • 当前帧和最近关键帧之间视角变化是否足够大。
  • 当前帧跟踪到的地图点数量是否下降。
  • 当前帧是否能提供足够新的场景区域。
  • 系统当前是否有能力处理新的关键帧。

关键帧太少，地图覆盖不足，跟踪容易失败。关键帧太多，后端优化压力大，冗余约束多。一个 SLAM 系统的稳定性，很大程度上取决于关键帧策略是否合理。

地图点生成和维护#

特征点法的地图通常由稀疏三维点组成。地图点可以通过双目深度、RGB-D 深度或单目多视角三角化得到。

单目三角化需要两个条件：一是同一个特征点在多个关键帧中被观测到，二是这些观测之间有足够视差。如果视差太小，深度估计会很不稳定。因此局部建图模块通常会检查视差、重投影误差、观测次数等指标，再决定是否保留一个地图点。

地图点也需要不断剔除。比如一个点被很少观测到，或者经常匹配失败，或者优化后重投影误差很大，就可能是错误点。保留太多错误地图点会污染后端优化，进一步影响位姿估计。

Bundle Adjustment#

Bundle Adjustment，简称 BA，是视觉 SLAM 后端优化的核心。它联合优化相机位姿和地图点位置，使所有观测的重投影误差最小。

可以把 BA 理解成这样一个问题：我们有很多相机位姿，也有很多三维地图点，每个地图点在若干关键帧中被观测到。现在这些位姿和点的位置都有误差，BA 要同时调整它们，让三维点投影到每张图像后尽可能对齐实际特征点。

BA 可以分为：

  • Motion-only BA：只优化当前帧位姿，地图点固定。常用于跟踪阶段。
  • Local BA：优化局部关键帧和局部地图点。常用于局部建图。
  • Global BA：优化所有关键帧和地图点。精度高但计算量大，通常在回环后或离线阶段使用。

BA 的计算代价很高，所以系统会利用稀疏结构和图优化库加速。传统视觉 SLAM 的后端本质上是一个稀疏非线性最小二乘问题。

回环检测#

视觉 SLAM 的前端和局部后端只能保证短期一致。随着时间变长，小误差会不断累积，轨迹会产生漂移。回环检测就是判断当前相机是否回到了过去到过的位置。

ORB-SLAM 系列常用词袋模型做地点识别。每张关键帧提取 ORB 描述子后，可以量化成视觉单词，形成一个词袋向量。当前关键帧和历史关键帧的词袋向量相似，就可能发生回环。之后还需要做几何验证，避免把相似但不同的地方误认为同一个地方。

一旦确认回环，系统会建立当前关键帧和历史关键帧之间的约束。这个约束告诉后端：这两个位置其实是同一个区域，轨迹应该闭合。

位姿图优化#

回环之后不能只修正当前帧，否则历史轨迹还是漂的。位姿图优化把关键帧看作图节点，把相邻关系和回环关系看作边。优化的目标是让所有边对应的相对位姿约束尽量一致。

位姿图优化通常比全局 BA 更轻量，因为它主要优化关键帧位姿，不直接优化所有地图点。优化后，轨迹的整体形状会被拉回一致状态。之后如果算力允许，可以再做全局 BA 进一步细化地图点。

特征点法的优点和局限#

整理下来，特征点法的优点还是很明显：

  • 实时性好，工程成熟。
  • 后端几何模型清晰，可解释性强。
  • 稀疏地图占用小，适合定位。
  • 回环和重定位方案成熟。

但它的问题也很直观：

  • 地图是稀疏的，不适合直接做高质量渲染或稠密场景理解。
  • 对弱纹理、重复纹理、运动模糊比较敏感。
  • 动态物体会造成错误匹配和错误地图点。
  • 单目尺度不确定，需要额外约束才能恢复真实尺度。

传统视觉 SLAM 先把定位、建图、优化这些问题讲清楚。把这套流程记住，再看更稠密、更鲁棒或者动态场景下的 SLAM 方法时，很多概念不会是空的。

记录#

传统特征点法的主线：提特征、做匹配、估计位姿、维护地图点、选择关键帧、局部 BA、回环检测、全局优化。它把视觉定位问题拆成了很清楚的几何模块。

不用一开始就陷在某个公式推导里。先把信息流理清楚更重要：图像如何变成特征观测，观测如何变成位姿约束，位姿和地图如何被联合优化，回环如何修正长期漂移。
